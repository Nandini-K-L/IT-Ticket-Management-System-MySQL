# IT Ticket Management System using MySQL

## Overview

The IT Ticket Management System is a SQL-based project developed using MySQL to manage IT service requests efficiently. It demonstrates relational database design and SQL querying through a realistic IT support ticket management scenario.

## Features

- Manage departments, users, and IT support tickets
- Create, update, and delete ticket records (CRUD operations)
- Track ticket status, priority, and assignments
- Analyze ticket data using business-oriented SQL queries
- Generate insights for operational reporting

## Database Structure

The project consists of three relational tables:

- **Departments** – Stores department information
- **Users** – Stores employee and support agent details
- **Tickets** – Stores IT support ticket information

## SQL Concepts Used

- DDL (CREATE DATABASE, CREATE TABLE)
- DML (INSERT, UPDATE, DELETE)
- SELECT Queries
- WHERE Clause
- ORDER BY
- GROUP BY
- HAVING
- DISTINCT
- Aggregate Functions (COUNT, AVG, MIN, MAX)
- INNER JOIN
- LEFT JOIN
- Subqueries
- CASE Statements
- LIKE
- BETWEEN
- IN / NOT IN

## Sample Business Queries

- Count tickets by status
- Count tickets by priority
- Count tickets by category
- Display open and closed tickets
- View department-wise ticket distribution
- Find agents handling the most tickets
- Search tickets using keywords
- Retrieve high-priority and critical tickets
- Display average tickets handled per agent
- Generate ticket statistics using CASE statements

## Technologies Used

- MySQL
- SQL
- MySQL Workbench

## Project Outcome

This project demonstrates practical SQL skills by designing a relational database and implementing business-oriented queries for IT ticket management. It showcases data retrieval, filtering, reporting, and analysis techniques commonly used in real-world database applications.
